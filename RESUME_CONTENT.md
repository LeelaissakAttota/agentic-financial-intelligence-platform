## Interview Talking Points

### "Walk me through the agent architecture."
> "The system uses a planner-executor pattern on LangGraph. A Planner Agent decomposes the query into a DAG of tasks. Three collection agents run in parallel: Financial Data (APIs), Web Research (search + extraction), Document Intelligence (SEC filings via LlamaIndex). Findings converge at an Analysis Agent (DCF, comps, scenarios). A RAG Agent retrieves grounded evidence using BGE-M3 + cross-encoder re-ranking. A Report Generator produces PDFs with inline citations. A Memory Agent stores everything in PostgreSQL+pgvector and Neo4j for future queries."

### "How do you handle hallucination in financial data?"
> "Three layers: 1) RAG grounding — every quantitative claim must have a retrieved chunk with citation. 2) Cross-encoder re-ranking ensures precision. 3) Structured output schemas (Pydantic) force the LLM to emit only validated fields. 4) Deterministic financial calculations (DCF, ratios) run in Python, not LLM. 5) Confidence scores propagated through the pipeline."

### "How does the memory system work?"
> "Three memory types: Episodic (full research sessions in PostgreSQL), Semantic (extracted facts/relationships in Neo4j knowledge graph), and Procedural (successful prompts/workflows). On new queries, the system retrieves relevant prior episodes and semantic facts, enabling few-shot prompting with actual prior work — the 10th report is faster and deeper than the 1st."

### "How do you evaluate agent quality?"
> "Multi-level: Unit tests for each agent (247 tests). Integration tests for full pipelines. Synthetic QA benchmarks for RAG retrieval (precision@k, citation accuracy). Financial calculation correctness tests. End-to-end golden-set evaluation: human-rated report quality on 50 historical analyses. Automated regression detection in CI/CD."
